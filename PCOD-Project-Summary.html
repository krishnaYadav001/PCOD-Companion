🌸 PCOD Companion

Project Summary & Design Documentation
A calm, intelligent Android wellness companion for women managing PCOD

The app observes patterns, offers gentle suggestions, and provides emotional support — designed to feel like a friend, not a clinical tracker.

1What the App Does

Daily wellness logging

Track everything that matters in one place — water, sleep, mood, food, exercise, stress, symptoms, medications, emotional state. Each field uses preset chips (≤2 taps to log) so the experience never feels like a form.

Personalized weekly plan

Build a routine of recurring tasks across Diet, Exercise, and Lifestyle categories. Tasks can be assigned to specific weekdays. A built-in generator creates a starter PCOD-friendly routine based on wake/sleep time, diet preference, and goal.

Cycle tracking

Log period start/end dates, flow level, and symptoms. Empty-state UI gently invites first entry.

Pattern detection (Insights)

Six observational patterns auto-detected from logged data — never prescriptive, always gentle:

  • Sleep ↔ Mood
  • Sugar ↔ Acne
  • Exercise ↔ Energy
  • Hydration ↔ Energy
  • Stress ↔ Sleep quality
  • Protein ↔ Mood

History & Summary

Browse past days as a calendar. Weekly summary with stats plus a doctor-export feature for sharing wellness reports.

4Engagement Features

Reward presence, not perfection

  • End-of-day closure card — first open of each day, gently summarizes yesterday based on logged data, non-judgmental at every completion level
  • Streak tracking without shaming on broken streaks
  • Soft failure feedback — empty states use floral language ("A fresh start ✨") never "no data" or "error"

Make every interaction feel light

  • Optimistic UI updates — checkboxes flip instantly, DB write happens in background
  • Haptic feedback on every state-mutating tap
  • Micro-interactions — press scale, accent border highlight, smooth list reorders, animated day-pill tabs
  • Snackbars not popups — rotating gentle messages ("Your week looks balanced 🌿")

Adaptive reminders

Three tiers (Low/Medium/High) for morning, hydration, evening check-ins, plus a daily re-engagement worker. Workers are smart — they skip if the user already engaged today, back off when ignored repeatedly (3+ ignores → every other day, 5+ → Mondays only), and reset when the user opens the app again.

Quiet Mode

A dedicated toggle that hides all suggestion cards (Focus, Insight, Plan Suggestion, Closure) and silences feedback snackbars. Logging and tracking remain unaffected. Reminder workers respect Quiet Mode — except the re-engagement nudge (intentional safety net).

First-launch guidance

  • Notification permission prompt with reasoning
  • First-insight helper card explaining what insights are
  • Empty-state CTAs throughout

Local backup & restore

JSON export/import via Storage Access Framework. Versioned format (currently BACKUP_VERSION = 3) for forward compatibility. Replace mode wipes existing data and inserts from backup; insights regenerate.

  • Press scale: 140ms tween, 1f → 0.97f
  • Highlight border: 200ms tween, accent at 55% alpha
  • List reorder/insert: 350ms tween via animateItemPlacement()
  • Color/progress tweens: 500–600ms
  • Spring scale on selection: spring(MediumBouncy, MediumLow)

Spacing & shape

Cards 16–22dp corners, chips 6–14dp · Vertical rhythm 4/8/12/16/24dp · 44dp minimum touch target · Pills via RoundedCornerShape(percent = 50)

Critical rule (we've shipped this bug twice — third time = unforgivable):
Cards with fixed light pastel backgrounds (Today's Focus, all category cards) must use the constant OnCategoryPastel (Color(0xFF2D2235)) for text — never MaterialTheme.colorScheme.onSurface, which flips to near-white in dark mode and becomes invisible.

7Database Discipline

Room strictly validates entity vs. DB schema after migration. Mismatched defaults crash the app on startup.

When adding a column:

  1. Add field to @Entity with @ColumnInfo(defaultValue = "...") matching migration SQL exactly
  2. Bump version in @Database(...)
  3. Add MIGRATION_OLD_NEW object in AppDatabase.companion
  4. Register in AppModule.provideDatabase() .addMigrations(...)
  5. Update BackupManager toJson/fromJson + bump BACKUP_VERSION if format changes

Conventions: Booleans → INTEGER "0" · Strings → TEXT "" · Numbers → INTEGER/REAL · Sentinel "not measured" → INTEGER "-1"

Current schema (v13)

  • daily_logs — date, mood, symptoms, water, sleep, exercise, fruit, sugar, stress, sleepQuality, protein, medications, streak, emotionalCheckIn, planCompletionPct (newest)
  • cycle_entries — startDate, endDate, flowLevel, symptoms, notes
  • plan_items — title, description, category, isCompleted, orderIndex, daysOfWeek
  • insights — key (unique), message, createdAt, lastShownAt, showCount

8Architecture Conventions

  • One activity, Compose-only. No fragments.
  • MVVM — each screen has XxxScreen + XxxViewModel (@HiltViewModel). State via MutableStateFlow<XxxUiState>.
  • Repository pattern — ViewModels never touch DAOs directly; go through PCODRepository / SettingsRepository.
  • One-shot UI events via MutableSharedFlow collected with LaunchedEffect.
  • Optimistic updates in TodayViewModel.updateLogOptimistic { ... }.
  • Quiet Mode chokepoint — TodayViewModel.emitFeedback() is the only way to fire feedback snackbars.
  • Reminder schedule chokepoint — WorkManagerHelper.applyReminderSchedule() is the single entry point for adaptive reminder workers.

10Things Worth Knowing (Gotchas)

TopicWhat to remember
AGP 9.x with compileSdk = 34Unusual combo — works but expect warnings. Don't bump compileSdk without checking other deps
WorkManager initApp implements Configuration.Provider and disables the default initializer in manifest. Don't re-enable it
scheduleDailyReset()Runs in Application.onCreate() — keep fast, no blocking work
Splash themeTheme.PCODCompanion.Splash extends Theme.SplashScreen. Don't change activity theme without updating both styles
Room destructive fallbackRebuilds the table — fine in dev, but does NOT help with column-default validation mismatches. Always match defaults exactly
Backup formatVersioned. Update toJson/fromJson AND bump BACKUP_VERSION if format changes incompatibly
FileProvider authority${applicationId}.fileprovider, paths in res/xml/file_paths.xml
Adaptive reminders single entryAlways go through WorkManagerHelper.applyReminderSchedule(). Workers themselves decide whether to fire
Re-engagement is exempt from Quiet ModeIntentional — it's the safety net for inactive users
Insights are derived dataWiped + regenerated on backup-import-replace and Reset All Data
Quiet Mode chokepointIn TodayViewModel, ALWAYS use emitFeedback(msg) — never _feedbackEvents.tryEmit directly
Plan completion tracking is forward-lookingPre-existing logs have planCompletionPct = -1. Real percentages accumulate going forward via the midnight worker. Day-history hint hides until ≥3 measured logs exist
Notification icon must be monochromeStatus bar tints small icons to a single color. Use ic_notification.xml (white-on-transparent leaf vector)

TL;DR

A production-ready Kotlin/Compose Android app for PCOD wellness companionship with 6 main screens, 13-version Room database, adaptive reminder system, JSON backup/restore, dark mode, Quiet Mode, persistent insights, end-of-day closure, and a fully codified design system. The defining quality is its emotional design — every interaction is calm, opt-in, and non-judgmental. Most recent work focused on the Plan Builder, which now has the most polished UI in the app and serves as the reference implementation of the design system.
